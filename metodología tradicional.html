Metodología tradicional

La metodología tradicional es un modelo de desarrollo de software donde las etapas del proyecto se realizan en un orden fijo y secuencial. También se conoce como modelo en cascada

Características

-Sigue un proceso ordenado.
-Cada etapa debe terminar antes de iniciar la siguiente.
-La documentación es muy importante.
-Los cambios durante el desarrollo son difíciles de realizar.

Etapas de la metodología tradicional

1. Análisis de requisitos
Se recopila toda la información del proyecto.
2. Diseño
Se planea cómo funcionará el sistema.
3. Desarrollo o programación
Se escribe el código del software.
4. Pruevas
Se detectan y corrigen errores.
5. Implementación
El sistema se entrega al usuario.
6. Mantenimiento
Se realizan actualizaciones y correcciones.
